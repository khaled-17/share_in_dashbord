# Share-In Application Documentation

## نظرة عامة (Overview)
تطبيق "Share-In" هو نظام إدارة مالي وإداري متكامل مصمم لتسهيل عمليات إدارة الشركات، بما في ذلك إدارة العملاء، الموردين، المصروفات، الإيرادات، والموظفين.

## التقنيات المستخدمة (Technology Stack)
- **Frontend Framework**: React 19 (Vite)
- **Language**: TypeScript
- **Styling**: Tailwind CSS
- **Routing**: React Router DOM v6
- **Database & Backend**: Supabase
- **Icons**: Heroicons (via SVG)
- **Notifications**: React Hot Toast

## هيكل المشروع (Project Structure)
```
src/
├── components/         # المكونات القابلة لإعادة الاستخدام
│   ├── layout/        # مكونات التخطيط (Sidebar, Header, MainLayout)
│   └── ui/            # عناصر واجهة المستخدم (Button, Card, Input, Table, etc.)
├── lib/               # مكتبات خارجية وإعدادات (Supabase client)
├── pages/             # صفحات التطبيق الرئيسية
│   ├── Dashboard/     # لوحة التحكم
│   ├── Customers/     # إدارة العملاء وتفاصيلهم
│   ├── Suppliers/     # إدارة الموردين وتفاصيلهم
│   ├── Expenses/      # إدارة المصروفات
│   ├── Revenue/       # إدارة الإيرادات
│   └── ...            # باقي الصفحات (Employees, Reports, etc.)
├── App.tsx            # المكون الرئيسي وإعدادات التوجيه (Routing)
└── main.tsx           # نقطة الدخول للتطبيق
```

## التوجيه (Routing)
يستخدم التطبيق `react-router-dom` للتنقل بين الصفحات. تم تكوين المسارات في ملف `App.tsx`.

### المسارات الرئيسية:
- `/`: لوحة التحكم (Dashboard)
- `/customers`: قائمة العملاء
- `/customers/:id`: تفاصيل العميل وسجل عملياته
- `/suppliers`: قائمة الموردين
- `/suppliers/:id`: تفاصيل المورد وسجل عملياته
- `/expenses`: إدارة المصروفات
- `/revenue`: إدارة الإيرادات
- `/payment-vouchers`: سندات الصرف
- `/receipt-vouchers`: سندات القبض
- `/quotations`: عروض الأسعار
- `/reports`: التقارير المالية

## قاعدة البيانات (Database Schema)
يعتمد التطبيق على Supabase كقاعدة بيانات Backend-as-a-Service. الجداول الرئيسية:

1.  **customers**: بيانات العملاء (الاسم، الهاتف، العنوان، إلخ).
2.  **suppliers**: بيانات الموردين (الاسم، التخصص، الهاتف، إلخ).
3.  **revenue**: سجلات الإيرادات (مرتبطة بالعملاء وأنواع الإيرادات).
4.  **expenses**: سجلات المصروفات (مرتبطة بالموردين وأنواع المصروفات).
5.  **revenue_types**: تصنيفات الإيرادات.
6.  **expense_types**: تصنيفات المصروفات.

## المميزات الرئيسية (Key Features)
1.  **إدارة العملاء والموردين**: إضافة، تعديل، وحذف بيانات العملاء والموردين مع إمكانية عرض سجل كامل لعملياتهم المالية.
2.  **تتبع المصروفات والإيرادات**: تسجيل دقيق للعمليات المالية مع ربطها بالعملاء/الموردين وتصنيفها.
3.  **الربط التشعبي**: إمكانية الانتقال من سجل الإيراد/المصروف مباشرة إلى ملف العميل/المورد.
4.  **واجهة مستخدم عصرية**: تصميم متجاوب وسهل الاستخدام يدعم اللغة العربية (RTL).

## كيفية التشغيل (How to Run)
1.  تأكد من تثبيت Node.js.
2.  قم بتثبيت الحزم:
    ```bash
    npm install
    ```
3.  قم بتشغيل خادم التطوير:
    ```bash
    npm run dev
    ```
4.  افتح المتصفح على العنوان الذي يظهر في التيرمينال (عادة `http://localhost:5173`).
